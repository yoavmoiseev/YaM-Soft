📚 What is School Auto Chat?

School Auto Chat is a fully offline classroom communication platform. It runs on the teacher's computer and everyone on the same local network (Wi-Fi or LAN cable) joins from their browser — no internet required, no installation needed for students.

FeatureDetails
💬 Real-time chatPublic classroom chat visible to everyone
🔒 Private messagesSend a message to one specific person
🤖 TeacherBotAI-style bot that searches your local knowledge base files
👨‍🏫 Admin roleTeacher gets full moderation controls
🌐 Works offlineNo internet connection required at any time
📱 ResponsiveWorks on laptops, tablets, and phones

🚀 Teacher Quick Start

Online version (Python installed)

  1. Open the Chat folder on your computer.
  2. Double-click run.bat — the server starts. You will see the local address in the console window.
  3. Open your browser and go to http://localhost:5002 (or the address shown in the console).
  4. Register with any username and the admin password (set in server.py, default: teacher123). You will automatically become the admin.
  5. Share your LAN IP address with students (shown in the console, e.g. http://192.168.1.5:5002).
✅ Tip: When Windows Firewall asks, click Allow access — this is required for students to connect over the network.

Offline version (no Python needed)

  1. Extract ChatApp-Offline.zip to any folder (USB drive, Desktop, etc.).
  2. Double-click START_CHAT.bat. A browser window opens automatically.
  3. The console shows your LAN address — share it with students.
ℹ️ The offline version includes Python 3.11 and all dependencies. Nothing needs to be installed on the teacher's PC.

🎒 Student Quick Start

  1. Connect to the same Wi-Fi or network as the teacher's computer.
  2. Open any browser and type the address the teacher shared, e.g. http://192.168.1.5:5002.
  3. Enter your name (or registered username) and a password. If you already registered, use the same credentials.
  4. You are in the classroom chat! Type in the message box and press Enter or click Send.
⚠️ You do NOT need to install anything. Just use your browser (Chrome, Firefox, Edge, Safari).

💬 Chat Features

Public chat

Type your message in the bottom input field and press Enter. Everyone in the room sees it.

Private message

Click a username in the Online Users list on the right side, then type your message. Only that person receives it.

Notifications

When you receive a private message, a badge appears on the user's name in the sidebar. The browser title will also show a notification indicator.

Online users list

The right panel shows all currently connected users. Icons indicate regular users vs. admin.

Message history

Up to the last 50 messages are shown upon joining (configurable in server.py).

🤖 TeacherBot Guide

TeacherBot searches your local knowledge_base/ files and returns the relevant passage. Send a message starting with @bot:

@bot Firewall
@bot IP address
@bot RAM

The bot responds in the public chat with the definition or passage found in your knowledge base.

How search works

BehaviorDetails
CaseCase-insensitive (firewall = Firewall = FIREWALL)
Match typeSubstring match — finds the term anywhere in a word or sentence
ResultReturns ~300 characters of context around the first match
No matchBot suggests the closest indexed words
Multiple filesSearches all .txt and .html files in knowledge_base/

Example interaction

Student: @bot protocol
TeacherBot: Found in IT_Glossary.txt:
  "Protocol — A set of rules that governs the communication
   between devices on a network. Examples: TCP/IP, HTTP, FTP."
✅ Best practice: Write each term once per file. If a term appears multiple times, only the first occurrence is returned.

💾 Offline Version (USB / LAN)

The offline package (ChatApp-Offline.zip) is fully self-contained: Python, Flask, and all dependencies are bundled inside. No internet and no installation required.

Structure of the ZIP

ChatApp-Offline/
├── START_CHAT.bat       ← double-click to start
├── launcher_offline.py  ← auto-detects port and opens browser
├── server.py            ← main chat server
├── python/              ← embedded Python 3.11 runtime
├── static/              ← CSS, JS, images
├── templates/           ← HTML templates
└── knowledge_base/      ← your .txt / .html resource files

Steps

  1. Extract ChatApp-Offline.zip to a local folder or USB drive.
  2. Double-click START_CHAT.bat.
  3. A browser tab opens automatically at http://localhost:5002.
  4. The console also shows your LAN address, e.g. http://192.168.1.5:5002 — share it with students.
ℹ️ The launcher automatically tries ports 5002, 5003, 5001, 8000, 8888, 5000 until a free one is found.
⚠️ Do not move or rename files inside the ZIP after extraction. The server uses relative paths.

📝 Knowledge Base Format

Place .txt or .html files in the knowledge_base/ folder. TeacherBot reads all of them at startup.

TXT format (recommended)

1. RAM (Random Access Memory) — Temporary storage used by the computer while running programs.
2. CPU (Central Processing Unit) — The main processor that executes program instructions.
3. Firewall — A network security system that monitors and controls incoming and outgoing traffic.

HTML format

Use simple tags only. The bot extracts visible text, ignoring CSS and scripts.

<p>Firewall — A network security system that monitors traffic.</p>
<p>IP Address — A logical address assigned to a network device.</p>

Best practices

  • Save files in UTF-8 encoding (important for non-Latin characters).
  • Write each term once per file for predictable results.
  • Use clear Term — Definition. format (dash or colon separator).
  • End sentences with periods for clean context extraction.
  • Avoid tables and complex HTML in .html knowledge base files.

🔧 Troubleshooting

ProblemSolution
Students can't connectMake sure Windows Firewall allowed the server. Check that devices are on the same network. Try ping <teacher-IP> from a student device.
Browser doesn't open automaticallyOpen it manually at http://localhost:5002 (offline) or http://localhost:5002 (online).
Server starts but shows errorsCheck the console window for the error message. Most common: port already in use — the launcher will try the next available port automatically.
TeacherBot says "not found"Check that the knowledge_base/ folder contains .txt or .html files and they are saved in UTF-8.
Garbled text / encoding issuesEnsure all knowledge base files are saved as UTF-8 (no BOM recommended for .txt).
Can't log in — wrong passwordUsers are stored in users_db.json. Ask the admin to delete and re-create the account if the password was forgotten.
