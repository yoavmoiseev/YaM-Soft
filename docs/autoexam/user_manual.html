3. Creating Exam Files

File Format Basics

Exams are created as simple .txt files using any text editor (Notepad, TextEdit, VS Code, etc.). The system automatically parses these files and converts them into interactive online exams.

Essential Rules

Rule 1: Question Numbering

Questions must start with a number followed by a dot and space:

‚ùå Wrong:

Question 1 - What is Python?
1 What is Python?
‚úÖ Correct: 
1. What is Python?
2. Which answer is correct?
3. Explain the concept.

Rule 2: Answer Placement

The FIRST answer is ALWAYS the CORRECT answer:

1. What is 2 + 2?
4          ‚Üê Correct answer (first)
5          ‚Üê Wrong answer
3          ‚Üê Wrong answer
6          ‚Üê Wrong answer
‚ö†Ô∏è Important: The system will automatically shuffle answers for students, so they won't see the correct answer first. You just need to place it first in your file.

Answers should NOT have numbers or bullets:

‚ùå Wrong:

6. Proctoring & Cheating Control üîí

What is Proctoring?

Proctoring means monitoring students during exams to ensure academic integrity. This system uses automatic digital proctoring - the computer continuously monitors student behavior and logs any suspicious activity without requiring a webcam or additional software.

Key Benefits:

  • Automatic Monitoring: No need to watch students manually
  • Detailed Logs: Every suspicious action is recorded with timestamps
  • Non-Intrusive: No webcam or screen recording required
  • Fair Testing: All students are monitored equally
  • Evidence-Based: Review concrete data about cheating attempts

How Protection Works

When a student starts an exam, the proctoring system automatically activates and monitors:

üëÅÔ∏è Focus Tracking

Detects when students leave the exam page or switch to other tabs/windows

üìã Copy Prevention

Blocks copying of questions and answers (Ctrl+C completely disabled)

üìù Paste Prevention

Prevents pasting from external sources (Ctrl+V blocked)

üñ±Ô∏è Right-Click Block

Disables context menu to prevent copying/inspecting

üõ†Ô∏è DevTools Block

Prevents opening browser developer tools and viewing source code

üîä Audio Warnings

Plays beep sounds when suspicious activity is detected

Detected Cheating Types

Event Type What It Detects Severity Student Impact
focus_lost Student switched to another tab/window/application Medium Warning beep plays
window_minimized Exam window was minimized or hidden Medium Logged silently
copy_blocked Attempted to copy text with Ctrl+C Medium Action blocked
paste_attempt Attempted to paste with Ctrl+V Medium Action blocked
right_click Right-clicked on exam page Low Context menu disabled
devtools_f12 Pressed F12 to open developer tools High Key press blocked
devtools_keyboard Pressed Ctrl+Shift+I/J/K (developer shortcuts) High Key press blocked
view_source Pressed Ctrl+U to view page source High Key press blocked
page_unload Attempted to close or reload the exam page Medium Logged + browser warning

What Students Experience

Students WILL Notice:

  • Warning beep sounds when they lose focus (switch tabs)
  • Copy/paste doesn't work (Ctrl+C and Ctrl+V disabled)
  • Right-click is disabled (context menu won't appear)
  • Developer tools won't open (F12 and shortcuts blocked)
  • Browser may show warning when trying to leave the page

Students WON'T See:

  • ‚ùå Pop-up messages about being monitored
  • ‚ùå Warnings when violations are logged
  • ‚ùå Count of how many times they've been caught
  • ‚ùå Real-time alerts about proctoring

Result: Monitoring is effective but non-intrusive

Reviewing Cheating Logs

After students complete exams, you can review all logged activity to identify potential academic dishonesty:

üìÅ Log Files

The system automatically stores logs in the logs/ folder:

  • cheating_alerts.log: All detected cheating attempts
  • exam_sessions.log: When students started exams
  • exam_submissions.log: Final scores and violation counts
  • login_history.log: Teacher and student login events

üìä Log Information Includes:

  • Student name and session ID
  • Exam ID and name
  • Type of cheating attempt
  • Exact timestamp (date and time)
  • Total violations per student
  • IP address and browser information

Interpreting Violations

üü¢ Low Concern (1-3 violations)

Likely causes:

  • Accidental tab switching due to notifications
  • Browser pop-ups or system alerts
  • Student unfamiliar with proctoring
  • Technical issues (slow computer/internet)

Action: Monitor but usually not a concern

üü° Medium Concern (4-10 violations)

Possible causes:

  • Student looking for answers in other tabs
  • Multiple attempts to copy questions
  • Trying to access external resources
  • Pattern of suspicious behavior

Action: Review logs carefully, may warrant a conversation

üî¥ High Concern (10+ violations)

Strong indicators of:

  • Deliberate cheating attempts
  • Persistent searching for answers
  • Trying to circumvent proctoring
  • Academic dishonesty

Action: Investigate thoroughly, follow school policy

Proctoring Limitations

‚ö†Ô∏è What Proctoring CAN Detect:

  • ‚úÖ Leaving exam window or switching tabs
  • ‚úÖ Copying/pasting attempts within the browser
  • ‚úÖ Right-clicking and trying to access source code
  • ‚úÖ Opening developer tools
  • ‚úÖ Minimizing exam window

‚ùå What Proctoring CANNOT Detect:

  • ‚ùå Using a mobile phone to search answers
  • ‚ùå Having notes on paper nearby
  • ‚ùå Someone else sitting next to the student providing help
  • ‚ùå Using a second computer or monitor
  • ‚ùå Taking photos of the screen with a camera
  • ‚ùå Having someone else take the exam

üí° Important: Digital proctoring is a helpful tool, but it's not perfect. Use it as one component of your exam security strategy, not as the only defense against cheating. Combine with other methods like:

  • Randomized question order
  • Time limits
  • Unique questions for each student
  • Honor code agreements
  • Follow-up oral examinations for suspicious cases

Privacy & Data Protection

‚úÖ What We DO Collect:

  • Student name (as entered by student)
  • Exam interaction data (clicks, focus events)
  • Keyboard events during exam (for proctoring only)
  • Timestamps of all activities
  • IP address and browser information

‚ùå What We DO NOT Collect:

  • ‚ùå Webcam images or video
  • ‚ùå Microphone or audio recordings
  • ‚ùå Screen recordings or screenshots
  • ‚ùå Personal files from student's computer
  • ‚ùå Browsing history outside the exam
  • ‚ùå Any personally identifiable information beyond name

Data Security: All logs are stored securely on your server and are accessible only to authorized teachers. Students cannot view or access proctoring logs.

9. Best Practices & Tips

Creating Effective Exams

‚úÖ DO:

  • Mix question types: Combine multiple-choice, open, and programming tasks
  • Test your exam: Use "Student Preview" before sharing with students
  • Provide clear instructions: Explain what students should do
  • Balance difficulty: Start with easier questions to build confidence
  • Use descriptive filenames: Easy to identify exams later
  • Keep backups: Save exam files on your computer
  • Review question quality: Ensure answers are clear and unambiguous

‚ùå DON'T:

  • Forget the correct answer order: Always place correct answer first
  • Use ambiguous questions: Make sure there's only one correct answer
  • Mix languages: Keep exams in a single language
  • Create overly long exams: Students may lose focus or run out of time
  • Skip testing: Always preview before deploying
  • Ignore formatting: Proper structure ensures correct parsing

Proctoring Best Practices

Before the Exam:

  • Inform students: Tell them proctoring is active
  • Explain rules: What's allowed and what's not
  • Technical check: Ensure students can access the system
  • Set expectations: Consequences for cheating
  • Practice run: Consider a practice exam first

During the Exam:

  • Be available: Monitor for technical issues
  • Check activity: See who's actively taking the exam
  • Respond to questions: Help with technical problems only
  • Don't intervene unnecessarily: Let proctoring work automatically

After the Exam:

  • Review logs promptly: Check proctoring data while it's fresh
  • Grade open questions: Don't delay manual grading
  • Investigate anomalies: Look into suspicious patterns
  • Follow policies: Apply your school's academic integrity rules
  • Provide feedback: Let students know their results

Exam Security Tips

üí° Enhance Security By:

  • Time limits: Set appropriate exam duration
  • Question pools: Randomize questions for different students
  • Unique exams: Create variations of the same exam
  • Scheduled access: Only allow exam access during specific times
  • One attempt policy: Prevent multiple submissions
  • IP restrictions: Limit access to specific locations (if needed)

Technical Tips

‚úÖ For Best Performance:

  • Save exam files as UTF-8 encoded text
  • Use consistent spacing and formatting
  • Test exams on different browsers
  • Keep exam files under 1MB in size
  • Use descriptive exam names
  • Organize exams by subject or date
  • Regular backup of important exams

Communication Tips

Communicating With Students:

  • Clear instructions: Provide written exam guidelines
  • Technical requirements: Tell students what they need
  • Exam link/code: Share in advance with clear deadline
  • Support contact: Provide way to reach you for help
  • Proctoring notice: Inform about monitoring in advance
  • Results timeline: When will they see their grades?

10. Troubleshooting

Common Issues & Solutions

‚ùì Exam Not Uploading

Possible causes:

  • File is not in .txt format ‚Üí Save as plain text
  • File is too large ‚Üí Keep under 1MB
  • Wrong encoding ‚Üí Save as UTF-8
  • Network issue ‚Üí Check internet connection

‚ùì Questions Not Displaying Correctly

Check formatting:

  • Questions must start with number + dot (e.g., "1.")
  • Answers should not have numbers or bullets
  • Correct answer must be first
  • Use "Exam Data" preview to see errors

‚ùì Proctoring Not Working

Possible causes:

  • Student using very old browser ‚Üí Update browser
  • JavaScript disabled ‚Üí Enable JavaScript
  • Browser extension interfering ‚Üí Disable extensions
  • Incognito/private mode ‚Üí Use normal browser mode

‚ùì Student Can't Access Exam

Check:

  • Is the exam link/code correct?
  • Is the exam activated and published?
  • Is exam scheduled for the current time?
  • Did student enter the correct code?
  • Network firewall blocking access?

‚ùì Excessive Focus Loss Events

May not indicate cheating if caused by:

  • Browser notifications or pop-ups
  • Operating system updates/alerts
  • Slow computer causing UI lag
  • Unstable internet connection
  • Antivirus or security software alerts

Getting Help

If problems persist:

  • Check system documentation and FAQs
  • Review log files for error messages
  • Test with different browser or device
  • Contact technical support with details:
    • What you were trying to do
    • What happened instead
    • Error messages (if any)
    • Browser and operating system
